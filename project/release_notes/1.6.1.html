qooxdoo 1.6.1 (2012-06-21)

Summary

This is a regular patch release, which includes many bugfixes but no feature enhancements over the previous release. It is fully backwards compatible to qooxdoo 1.6. Nothing needs to be changed in the JavaScript code of your existing apps if they are based on the previous 1.6 release.

Bugfixes

See the corresponding bugzilla query for more than 110 issues resolved for qooxdoo 1.6.1.

Framework

Core

  • Fixed Array handling in qx.lang.Object.toUriParameter.
  • Replace IE's incomplete Error.toString with qx.lang.Core.errorToString.
  • Fixed event data on qx.data.Array (#6154)
  • Removed warning for images loaded via data url.
  • Commands don't override themed icons anymore. (#5906)
  • Serializer processes inherited properties now. (#5559)
  • If an assertion check throws an error, the message will include responsible arguments with value of undefined as 'undefined' (#6282)

Data Binding

  • Fixed support for multiple mixins for the JSON marshaler. (#6445)
  • sort() method of qx.data.Array was firing events (change, changeBubble) although nothing has been changed on the array. In this case we are now preventing these events.(#6213)

Browser Support

  • Feature detection for CSS border-image syntax to cope with differences between prefixed and unprefixed/spec-compliant implementations.
  • Fixed animation for IE10.
  • Fixed gradient calculation for IE10. (#6513)

Low-level

  • Added support for IE8 to qx.bom.IframeHistory. Even though IE8 can natively use "hashchange" events, it fails if qooxdoo runs in an iframe. Therefore we need to poll a helper iframe like in in IE6/7 (#6194)(#6521)

UI Core

  • Improved overall scroll handling.
  • Adding of any object which is not a qx.ui.core.LayoutItem throws now an error which points you to that issue (#6281)

Form

  • Form now supports the virtual select box. (#5711)

Widgets

  • Disabled resizing for disabled widgets e.g. windows. (#6516)

Virtual Widgets

  • Fixed infinity loop for one selection when the last item from the VirtualList is removed. (#6340)
  • Nested RadioGroupBox are now correctly disabled when parent is disabled. (#5445)

Theming

  • Make themed fonts aware of colors from the color theme. (#6496)

Mobile

  • Fixed Android OS detection (#5971)

Tooling

Internationalization

  • The Norwegian language code no has been removed from the CLDR data. Please use nb ("Norwegian Bokm√•l", which "no" used to redirect to) or nn ("Norwegian Nynorsk") instead. "NO" as territory code is ok (as in "nb_NO").
